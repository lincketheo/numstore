WAL Notes:

Non Fuzzy Checkpoints
	1 Stop all new txns from starting 
	2 Let current txns complete
	3 Take a Checkpoint

	- Easy recovery
	- Dependent on slow txns

ATT:
	1 Per Txn:
		txid 
		status (Running, Committing, Candidate for Undo)
		lastLSN - Most recent lsn created by txn

DPT:
	1 Per Dirty Page:
		pgno
		recLSN - LSN of the log record that first caused the page to be dirty

Fuzzy Checkpoint:
	- CHECKPOINT BEGIN (no parameters)
	- Active txns can continue to run while system writes the log records 
	- CHECKPOINT END
			serialized att
			serialized dpt
